# Implementation Plan: Issue #62 -- Improve Test Coverage

## Overview

10 tasks across 9 modules. All test-only -- no production code changes.
Tests use Vitest with `happy-dom` environment. Existing patterns from `disk-cache.test.ts` and `format.test.ts`.

**Conventions observed:**
- Test files co-located next to source (D1)
- `vi.mock('node:fs')` for filesystem tests (D2)
- `vi.resetModules()` + dynamic import for module-level constants like `CLAUDE_DIR` (D3)
- `vi.useFakeTimers()` for time-dependent functions
- Run individual test: `cd apps/web && npx vitest run --reporter verbose <path>`

import { renderHook, act } from '@testing-library/react'
import { ThemeProvider, useTheme } from './ThemeProvider'

function wrapper({ children }: { children: ReactNode }) {
  return <ThemeProvider>{children}</ThemeProvider>
}

beforeEach(() => {

  document.documentElement.removeAttribute('data-theme')
